## Emmet 写法

Emmet是一套高效的HTML/CSS代码缩写语法，能让你用类似“**代码快捷键**”的方式快速生成结构化代码

输入简写元素名或符号，再按回车即可快速生成代码结构

### HTML的Emmet写法

| Emmet                                        | 生成结果                                                     |
| -------------------------------------------- | ------------------------------------------------------------ |
| **标签名**，自动生成标签，并补全常用属性     | <标签名 属性=“”></标签名>                                    |
| div                                          | `<div></div>`                                                |
| img                                          | `<img src="" alt="">`                                        |
|                                              |                                                              |
| **标签名.类名**，生成标签和class属性         | <标签名 class=“类名”></标签名>                               |
| div.text                                     | `<div class="text"></div>`                                   |
| img.text                                     | `<img src="" alt="" class="text">`                           |
| **标签名#id名**，生成标签和id属性            | <标签名 id=“id名”></标签名>                                  |
| div#text                                     | `<div id="text"></div>`                                      |
| img#text                                     | `<img src="" alt="" id="text">`                              |
|                                              |                                                              |
| **标签名1+标签名2**，兄弟元素，同级元素      | <标签名1></标签名1><br /><标签名2></标签名2>                 |
| div+p                                        | `<div></div>`<br />`<p></p>`                                 |
| **标签名1>标签名2**，子级元素                | <标签名1><br />      <标签名2></标签名2><br /></标签名1>     |
| div>p                                        | `<div>`<br />     `<p></p>`<br />`</div>`                    |
| 标签名1>标签名2**^标签名3**，跳出嵌套        | <标签名1><br />      <标签名2></标签名2><br /></标签名1><br /><标签名3></标签名3> |
| div>p^img                                    | `<div>`<br />     `<p></p>`<br />`</div>`<br />`<img src="" alt="">` |
|                                              |                                                              |
| **标签名\*n**，生成n个同名标签               | <标签名></标签名>……                                          |
| li*3                                         | `<li></li>`<br />`<li></li>`<br />`<li></li>`                |
|                                              |                                                              |
| **标签名{文本内容}**，生成标签并插入文本内容 | <标签名>文本内容</标签名>                                    |
| div{文本内容}                                | `<div>文本内容</div>`                                        |

# HTML 基础语法

## 标签

HTML特有的语法格式，有单标签和双标签两类

1. 双标签：`<strong>……</strong>` 加粗字体
2. 单标签：`<br>` 换行

## 属性与属性值

属性写在尖括号里面，标签名后面，用来控制标签的行为，属性与属性间用**空格**隔开

语法：属性名 = “属性值”

示例：`<img src="./cat.jpg" alt="替换文字">`

## 注释

写在 `<!--` 和 `-->` 之间的内容不会显示在结果页面上，用于程序员添加代码注释

## HTML基本框架介绍

```html
<!DOCTYPE html>
<html lang="zh-CN">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Document</title>
</head>
<body>
    网页内容
</body>
</html>
```

1. **`<!DOCTYPE html>`**
	* **声明文档类型**：它告诉浏览器当前文档使用的是 HTML5 规范。虽然它不是 HTML 标签，但必须放在文档的第一行。如果不写，可能会导致页面渲染出现兼容性问题。
2. **`<html>`**
	* **根元素**：这是整个 HTML 文件的容器，所有其他的标签都必须写在它内部。
	* **属性 `lang="zh-CN"`**：告诉浏览器这个页面的主要语言是简体中文。
3. **`<head>`**
	* **头部容器**：用于存放页面的**元数据**（Metadata）。这些内容通常不会直接显示在网页的正文区域，但对浏览器的渲染和搜索引擎至关重要。
	* **常见子标签**：
		* `<meta>`：定义字符编码、视口设置、页面描述等。
		* `<title>`：定义浏览器标签页上显示的标题。
		* `<link>`：引入 CSS 样式表或图标。
		* `<script>`：引入或编写 JavaScript 代码。
4. **`<meta charset="UTF-8">`**
	* **设置字符编码**：告诉浏览器这个文件使用的是 UTF-8 编码。这非常重要，否则如果你的网页中包含中文，可能会显示为乱码。
5. **`<meta name="viewport" content="...">`**
	* **设置视口**：主要用于移动端适配，如果没有这行代码，在手机上打开网页时，文字通常会变得很小，需要手动放大才能看清。
	* **属性解释**：
		* `width=device-width`：让网页宽度等于设备屏幕宽度。
		* `initial-scale=1.0`：设置初始缩放比例为 1.0（不缩放）。
6. **`<title>`**
	* **设置网页标题**：会显示在浏览器的标签页标题栏上，也是用户收藏网页时默认的名称。
7. **`<body>`**
	* **主体容器**：这里包含了所有用户在浏览器窗口中能看到的内容。比如文本、图片、链接、按钮、视频等，都必须写在 `<body>` 标签内部。

## 相对路径和绝对路径

功能：插入多媒体元素的路径格式，相对路径应用面更广泛且可移植性好，绝对路径在网页链接中用的多

1. 相对路径：从当前文件位置出发查找目标文件

	* /	表示**进入**某个文件夹里面

	* .     表示当前文件所在**文件夹**

	* ..    表示当前文件的**上一级文件夹**

2. 绝对路径：从盘符[^3]出发查找目标文件
    在写路径时，要注意Win系统支持两种斜杠（\，/）；其他系统仅支持左斜杠（/），为了统一和可移植性，建议以后默认使用左斜杠（/）

示例：

```html
<!-- 当前文件夹下 -->
<img src="./甘雨-祈祷.png">

<!-- 下一级文件夹下 -->
<img src="./相对路径/甘雨-打瞌睡.png">

<!-- 上一级文件夹下的另一子文件夹下 -->
<img src="../相对路径/甘雨-挺好的.png">

<!-- 绝对路径 -->
<img src="D:/Pictures/portrait1/方形素材 (1).jpg" width="200">
<img src="https://www.miyoushe.com/_static/img/miHoYo_Game.2457753.png">
```

## 超链接标签

功能：插入超链接

语法：

```html
<a href="链接地址" target="打开方式" title="显示文本">链接名称</a>
```

属性解释：

* herf：指定链接的目标地址，常见的有以下类型
	* **跳转到其他网址**：`href="https://www.bilibili.com/"`
	* **跳转到本页面内指定位置**：`href="#section1"`
	* 跳转到本地文件：`href="./相对路径/甘雨-打瞌睡.png"`
	* 打开邮件客户端：`href="mailto:test@example.com"`
	* 拨打电话（移动端）：`href="tel:+8613800000000"`
	* **空链接**：`href="#"`
* target：控制链接在何处打开，有以下属性值
	* _self：默认值，在当前窗口打开，会覆盖当前窗口
	* _blank：在新窗口打开
* title：鼠标悬停在链接上时显示的提示信息

示例：

```html
<!-- 跳转到其他网页上 -->
<a href="https://www.bilibili.com/">bilibili视频弹幕网</a>

<!-- 跳转到本地文件上 -->
<a href="./相对路径/甘雨-打瞌睡.png">甘雨-打瞌睡</a>

<!-- 设置空链接 -->
<a href="#">空链接</a>
```

## 表格

功能：生成一个表格，默认无边框线，可使用 **border 属性**为表格添加边框线

语法：

```html
<table border="1">
    <tr>
        <th>表头1</th>
        <th>表头2</th>
        <th>表头3</th>
    </tr>
    <tr>
        <td>单元格1</td>
        <td>单元格2</td>
        <td>单元格3</td>
    </tr>
    <tr>
        <td>单元格1</td>
        <td>单元格2</td>
        <td>单元格3</td>
    </tr>
    <tr>
        <td>单元格1</td>
        <td>单元格2</td>
        <td>单元格3</td>
    </tr>
</table>
```

属性解释：

* border：放在 table 标签里，添加表格框线，取值是数字，单位为像素
* rowspan：放在 th 或 td 标签里，合并行单元格，取值是数字，意为合并几行
* colspan：放在 th 或 td 标签里，合并列单元格，取值是数字，意为合并几列

### 行合并和列合并的注意事项

* 合并单元格的步骤：
	1. 明确合并的目标
	2. 保留**最左最上**的单元格，添加属性（取值是数字，表示需要合并单元格的数量）
	3. 删除其他被合并的单元格
* tr 是行，th、td 是列，行嵌套列，其中 th 是表头标签，带有加粗效果
* 了解表格结构标签：下面3个标签是用来对浏览器说明表格的格式划分，默认就有，故仅了解
	* `<thead>...</thead>`：表格头部，包裹表头
	* `<tbody>...</tbody>`：表格主体，包裹主体
	* `<tfoot>...</tfoot>`：表格底部，包裹底部
* 行合并和列合并都不能跨结构标签合并

### input 标签

功能：**文本框、密码框、单选框、多选框和上传文件**

语法：

```html
文本框：
<input type="text" placeholder="提示信息">
<br><br>
单选框：
<input type="radio" name="gender" checked>男

多选框：
```

	* password：密码框
	* radio：单选框，并不是真正意义上的单选，需要进一步设置属性实现
	* checkbox：多选框
	* file：上传文件，默认只能上传一个文件
* placeholder：**文本框和密码框**可以使用，设置框内提示信息
* name：**单选框和多选框**可以使用，设置控件名称，同组同时只能选一个（**单选功能**）
* checked：**单选框和多选框**可以使用，设置默认选中状态，单属性
* multipe：**上传文件**可以使用，实现上传多个文件功能

### label 标签

功能：一是显示控件说明文本；二是绑定文本和控件的关系，**扩大表单控件点击区域**，提升用户体验

语法：

```html
写法一：label标签只包裹内容，不包裹表单控件；设置label标签的for属性值和表单控件的id属性值相同
<input type="radio" id="man">
<label for="man">男</label>

写法二：使用label标签包裹文字和表单控件，不需要属性
<label><input type="radio">女</label>
```

label支持扩大控件点击区域，对以下对象有效：**文本框、密码框、单选框、多选框、上传文件、下拉菜单和文本域**等等

1. 一个label对应一个控件，不能一个label包含多个控件。
2. 扩大控件点击区域的功能主要用在单选框和多选框上。

属性解释：

* id：给 `<input>` 元素起一个全局唯一的名字，相当于一个锚点

## 基础选择器

选择器的作用是查找标签，设置样式，基础选择器有**标签选择器、类选择器、id选择器和通配符选择器**，生效优先级：==**id > 类 > 标签 > 通配符**==，开发推荐使用类选择器

| 选择器       | 作用范围           | 复用性                     | 优先级 |
| ------------ | ------------------ | -------------------------- | ------ |
| 标签选择器   | 同名标签           | 可复用                     | 低     |
| 类选择器     | 引用该类的标签     | 可复用可叠加[^6]           | 中     |
| id选择器     | 唯一引用该id的标签 | 不能复用，一个id只能用一次 | 最高   |
| 通配符选择器 | 所有标签           | 不能复用                   | 最低   |

### 标签选择器

语法：用**标签名**作为选择器，对页面内所有同名标签应用CSS样式，缺点是过于统一化，DIY度不高

```html
/*将页面中所有p标签中的文本设置成红色*/
<style>
    p {
        color:red;
    }
</style>
```

### 类选择器

语法：自定义**选择器名称（类名）**，差异化设置每个标签的显示效果

1. 定义选择器（定义**`.类名`**），取类名时，尽量取得见名知义
2. 使用类选择器，标签添加 **class属性**，`class="类名"`

一个类选择器可以被多个标签使用；一个标签也可以使用多个类选择器，引用时，多个类名之间用**空格**隔开

```html
<style>
    /*定义类选择器，注意格式是 .类名*/
    .red {
        color:red;
    }
    .size {
        font-size:50px;
    }
</style>

<!--使用类选择器，注意格式是 类名，不加点-->
<p class="red">这是红色文字</p>
<!--引用多个类名，类名之间用空格隔开-->
<div class="red size">
    这是一个块，块内文字是红色，字体大小是50像素
</div>
```

### id选择器

语法：自定义**选择器名称（id名）**，差异化设置每个标签的显示效果，id选择器更经常用来配合JavaScript使用，很少用来设置CSS样式

1. 定义id选择器（定义 `#id名`），id名自定义，尽量取得见名知义
2. 使用id选择器，标签添加 **id属性**，`id="id名"`

与类选择器的不同点：

1. 在一个.html文件中，每个标签使用的id值必须是**唯一**[^7]的，**不能重复引用**
2. 一个标签只能使用一个id选择器，不能引用多个
3. 一个标签可以同时引用类选择器和id选择器，但必须遵守它们各自的引用规则

```Html
<style>
    /*定义id选择器，注意格式是 #id名*/
    #red {
        color:red;
    }
</style>

<!--使用id选择器，注意格式是 id名，不加#-->
<p id="red">这是红色文字</p>
```

### 通配符选择器

语法：对页面内所有标签设置统一CSS样式

定义通配符选择器（星号`*`），浏览器自动查找页面所有标签，应用统一样式

```html
<style>
    * {
        color:red;
    }
</style>
```

### 字体族（字体选择）

功能：设置所使用的字体

1. `font-family`属性值可以添加多个字体名，用**逗号**隔开，浏览器会按**从左至右**的优先级在系统上查找字体并生效

2. 不要害怕，-apple-system, system-ui, BlinkMacSystemFont, 'Segoe UI', Roboto, 'Helvetica Neue', Arial, sans-serif 这些都是字体名，可以根据下面的表格了解一下

	| 字体名称             | 解释                                                         |
	| -------------------- | ------------------------------------------------------------ |
	| `-apple-system`      | macOS和iOS的系统默认字体（San Francisco）                    |
	| `system-ui`          | 标准CSS关键字，代表当前操作系统的UI默认字体（现代浏览器支持，可跨平台） |
	| `BlinkMacSystemFont` | Chrome/Opera在macOS上识别San Francisco字体的旧方式（兼容性） |
	| `'Segoe UI'`         | Windows系统的默认UI字体                                      |
	| `Roboto`             | Android和Chrome OS的默认字体                                 |
	| `'Helvetica Neue'`   | 较老版本macOS的经典字体（在San Francisco之前）               |
	| `Arial`              | 几乎所有系统都有的经典无衬线字体（通用后备）                 |
	| `sans-serif`         | 通用字体族关键字，告诉浏览器：“随便选一个无衬线字体”         |

3. 关于引号问题：单词字体名（如`Arial`）不用加引号，多词字体名（如`'Segoe UI'`）必须加引号
4. `system-ui`是CSS标准定义的关键字，理论上可以只写它一个，但为了兼容老浏览器，还是保留完整的字体栈更安全
5. `sans-serif`通常是`font-family`属性最后设置的字体族名，表示无衬线字体族，用来保底

## 复合选择器

由多个基础选择器组合而成，更准确、更高效的选择目标元素，有**后代选择器、子代选择器、并集选择器和交集选择器**，它们的区别主要在于元素之间的层级关系和组合方式

| 选择器     | 写法    | 含义                  |
| ---------- | ------- | --------------------- |
| 后代选择器 | `A B`   | A内部所有B，包括B以下 |
| 子代选择器 | `A > B` | A的直接子级B          |
| 并集选择器 | `A, B`  | A 或 B                |
| 交集选择器 | `.A.B`  | A 且 B                |

### 后代选择器

功能：选中当前元素的所有后代元素（无论嵌套多深），应用CSS样式

语法：`父选择器 子选择器 {CSS属性}`，父子选择器用**空格**隔开

```html
<style>
    <!--选中div的所有后代标签，应用样式-->
    div p {
        color:red;
    }
</style>

<div>
  <p>div的儿子</p>
  <section>
    <p>div的孙子</p>
  </section>
</div>
```

结果：两个`<p>`都会变红，因为它们都是`div`的后代

### 子代选择器

功能：选中当前元素的所有直接子级元素（儿子一级），应用CSS样式

语法：`父选择器>子选择器 {CSS属性}`，父子选择器用 **>** 隔开

```
<style>
    <!--选中div的所有子代标签，应用样式-->
    div > p {
        color:red;
    }
</style>

<div>
  <p>div的儿子</p>
  <section>
    <p>div的孙子</p>
  </section>
</div>
```

结果：只有第一个`<p>`变成红色，第二个不变，因为它不是`div`的直接子级元素

### 并集选择器

功能：选中多个元素设置相同的CSS样式，常用于写一些标签的共同属性

语法：`选择器1, 选择器2,...， 选择器n {CSS属性}`，选择器之间用**逗号**隔开

```html
h1, h2, h3 {
  font-weight: bold;
  margin-bottom: 10px;
}
```

结果：所有 `<h1>`、`<h2>`、`<h3>` 都会加粗并设置下边距

* 并集选择器不是“组合选择器”，它只是把多个选择器的结果合并，**不涉及元素间的关系**[^8]

### 交集选择器（了解）

功能：选中同时满足多个条件的元素，应用CSS样式

语法：`选择器1选择器2 {CSS属性}`，选择器连写，中间没有任何符号，注意，如果交集选择器中有标签选择器，标签选择器必须写在最前面

```html
div.highlight { ... } /* 是 div 且有 class="highlight" */
#nav.active { ... }   /* id="nav" 且有 class="active" */
```

* `.box .active`（有空格） VS `.box.active`（无空格），要搞清它们的区别
	前者表示`.box` 里面的 `.active` 元素（后代）
	后者表示同一个元素既有 `box` 类又有 `active` 类（交集）

## 伪类选择器

**伪类**表示元素状态，选中当前元素的某个状态（如鼠标悬停时状态）设置CSS样式

鼠标悬停时状态：`选择器:hover{CSS属性}`，任何标签都可以设置鼠标悬停状态

### 伪类-超链接

| 伪类       | 解释           |
| ---------- | -------------- |
| `:link`    | 访问前状态     |
| `:visited` | 访问后状态     |
| `:hover`   | 鼠标悬停时状态 |
| `:active`  | 点击时状态     |

1. 如果要给超链接设置以上4个状态，必须按顺序（**LVHA**）书写，否则可能不生效
2. 开发中，一个 a 标签选择器设置超链接样式，hover 状态单独设置，很少用到其他状态

## CSS三大特性

### 继承性

子级默认继承父级的**文字控制属性**[^9]，其他属性则不会继承

```html
<div style="color: red; font-size: 20px;">
  父元素
  <p>子段落</p>  <!-- 自动变红、20px -->
</div>
```

`<p>`没写样式，但**继承了**父级的文字颜色和字号

> 注意，继承的优先级**很低**，如果子级标签自己有样式（无论是浏览器默认样式，还是你写的CSS），会优先生效自己的样式，此时不发生继承
>
> 核心原则：**“显式声明 > 继承”**，CSS的继承机制是“后备方案”——只有当子元素**自己没有设置某个可继承属性**时，才会去拿父元素的值，一旦子元素自己设置了该属性（哪怕只是浏览器默认样式），就不再继承

### 层叠性

当多个CSS规则作用于同一个元素时，浏览器需要决定哪个规则最终生效——这个过程叫“层叠”

* 相同的属性会覆盖：后面的CSS属性会覆盖前面的CSS属性

* 不同的属性会叠加：不同的CSS属性会叠加，都生效

```html
<style>
    div {
        color:red;
        font-weight:700;
    }
    div {
        color:green;
        font-size:30px;
    }
</style>
```

`div` 最后的样式是字体绿色（相同：后覆盖前），加粗、字体大小30像素（不同：一起叠加生效）

### 优先级

优先级也叫**权重**，当一个标签使用了多种选择器时，基于不同类别的选择器的匹配规则，优先级高的越先生效

1. 通配符选择器 < 标签选择器 < 类选择器 < id选择器 < 行内样式 < !important（**选中标签范围越大，优先级越低**）

2. `!important`是CSS的一个关键字，可以强制提高某条CSS样式的权重到**最高**，但**请慎用**
	语法：`选择器{属性:属性值 !important;}`，写在属性值后面、分号前面，不是作用在整个规则块上，而是针对于某条CSS属性

3. 继承的优先级最低

4. 复合元素优先级计算规则（四元组计算）

	| 选择器       | 权重值           |
	| ------------ | ---------------- |
	| 行内样式     | (1,0,0,0)        |
	| id选择器个数 | (0,1,0,0)        |
	| 类选择器     | (0,0,1,0）       |
	| 标签选择器   | (0,0,0,1)        |
	| 通配符/继承  | (0,0,0,0) 无权重 |

	按上面的表格，从上至下依次比较选择器个数，同一级个数多的优先级高，如果个数相同，则继续向下比较

## 背景属性

### 背景色

功能：设置元素的背景颜色

语法：background-color（**bgc**）

```css
div {
  	background-color:#f0f0f0; /* 浅灰色 */
}
```

* 属性值可以是**关键词、rgb、rgba或十六进制**格式，这是颜色属性通用的格式，以后不再赘述

### 背景图

功能：设置元素的装饰性图片效果，请不要被背景图的名字误导了，它的功能不止做背景

语法：background-image（**bgi**）

```css
div {
  	background-image:url(./images/1.png);
}
```

1. 如果同时设置了背景色和背景图，那么背景图会占据上方，背景色则作为最底层
2. 背景图默认以**平铺**[^10]的效果填充元素区域，多余部分会被裁剪

### 背景图平铺方式

功能：控制背景图是否重复以及如何重复

语法：background-repeat（**bgr**）

```css
div {
  	background-image:url(./images/1.png);
    background-repeat: no-repeat;
}
```

| 属性值    | 解释         |
| --------- | ------------ |
| no-repeat | 不平铺       |
| repeat    | 平铺（默认） |
| repeat-x  | 水平方向平铺 |
| repeat-y  | 垂直方向平铺 |

### 背景图位置

功能：控制背景图在元素区域中的初始位置

语法：background-position（**bgp**）

```css
div {
  	background-image:url(./images/1.png);
    background-position:0 0;
}
```

| 属性值              | 解释          |
| ------------------- | ------------- |
| left                | 左侧          |
| right               | 右侧          |
| center              | 居中          |
| top                 | 顶部          |
| bottom              | 底部          |
| 数字+px，正负都可以 | 横坐标 纵坐标 |
| 百分比（0%~100%）   | 横坐标 纵坐标 |

1. 背景图默认位置是`0 0`（左上角）
2. 关键字取值方式可以不考虑横纵坐标先后书写顺序；数字和百分比取值方式，第一位是横坐标，第二位是纵坐标
3. 数字取值方式可以取正负数，**正数**表示**向右或向下**，**负数**表示**向左或向上**
4. 关键字、数字和百分比取值方式可以混用
5. 可以只写一个关键字，另一个方向默认居中；数字只写一个表示水平方向，垂直方向默认居中

### 背景图缩放

功能：设置背景图大小

语法：background-size（**bgz**）

```css
div {
  	background-image:url(./images/1.png);
    background-size: contain;
}
```

| 属性值                  | 解释                                                         |
| ----------------------- | ------------------------------------------------------------ |
| cover                   | 等比例缩放背景图，使其**完全覆盖**整个容器区域，图片可能会被裁剪 |
| contain                 | 等比例缩放背景图，使其**完整显示**在容器内，容器可能会有空白区域 |
| 百分比（0%~100%）       | 以容器宽度和高度的百分比来设置背景图尺寸，非等比缩放         |
| 数字+单位（如：px、em） | 使用具体单位指定宽高，非等比缩放                             |

### 背景图固定

功能：当页面滚动时，背景图像保持不动

语法：background-attachment（**bga**）

```css
div {
  	background-image:url(./images/1.png);
    background-attachment: fixed;
}
```

| 属性值 | 解释                                       |
| ------ | ------------------------------------------ |
| fixed  | 上下滚动页面时，背景图像会保持在相同的位置 |

### 背景图复合属性

功能：设置背景图公共样式，是一个**复合属性**，各个属性值用**空格**隔开，不区分先后顺序

语法：background（**bg**）

```css
div {
    /*background:背景色 背景图 背景图平铺方式 背景图位置/背景图缩放;*/
    background: #000 url(./images/1.png) no-repeat right center/cover;
}
```

1. `background:背景色 背景图 背景图平铺方式 背景图位置/背景图缩放;`，顺序不强制，但推荐按例子顺序来

## 显示模式

元素的显示方式，如元素在页面中如何呈现，是否独占一行，能否设置宽高、内外边距等

| 块级元素                     | 行内元素                          | 行内块元素           |
| ---------------------------- | --------------------------------- | -------------------- |
| 独占一行                     | 一行可显示多个                    | 一行可显示多个       |
| 宽度默认是父级的100%         | 宽高尺寸由内容撑开                | 宽高尺寸可由内容撑开 |
| 设置宽高属性生效             | 设置宽高属性不生效                | 设置宽高属性生效     |
| 例如：`<div>`, `<p>`, `<h1>` | 例如：`<span>`, `<a>`, `<strong>` |                      |

### 转换显示模式

功能：改变元素的默认显示行为（如将行内元素变为块级元素，或将块级元素并排显示等），从而实现灵活的页面布局

语法：

```css
img {
    display:block ;/*img元素会变成块级元素，独占一行*/
}
```

| 属性值       | 解释   |
| ------------ | ------ |
| block        | 块级   |
| inline-block | 行内块 |
| inline       | 行内   |
